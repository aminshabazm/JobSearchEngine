  WHAT THIS SYSTEM DOES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  An autonomous job-hunting pipeline that can be triggered on demand (or run
  daily via Windows Task Scheduler). It finds remote Software Engineering jobs
  in the USA, scores each one against your resume using Groq AI (free, cloud,
  <1 second per job), drafts personalised outreach emails for the best matches,
  and presents everything in a web UI where you review and approve sending.

  Cost: $0/month. You own the data. Email only sends when YOU click Approve.

  BACKEND — PYTHON PIPELINE
  ─────────────────────────
  Language        : Python 3.11+
  HTTP Client     : requests 2.31+  — calls the Indeed Scraper API
  AI / LLM        : groq 0.9+       — Python client for Groq cloud API
  Web Framework   : FastAPI 0.111+  — REST API that powers the React frontend
  ASGI Server     : uvicorn 0.29+   — runs FastAPI (development + production)
  Database        : SQLite 3        — built into Python stdlib, zero setup
  Email           : smtplib         — Python stdlib, sends via Gmail SMTP SSL
  Logging         : logging         — Python stdlib, rotating file + console
  Form Parsing    : python-multipart — required by FastAPI

  AI — GROQ API (CLOUD, FREE)
  ────────────────────────────
  Provider        : Groq Cloud — https://console.groq.com
  Model           : llama-3.3-70b-versatile  (70B params — far better than local)
  Free tier       : 14,400 requests/day — you use ~80/day (0.5% of limit)
  Speed           : <1 second per job (vs 15-30 sec with local CPU Ollama)
  Temperature     : 0.3 for scoring, 0.6 for email drafting
  Response format : JSON mode (response_format={"type": "json_object"})
  Use             : Score jobs 1–10 against resume, draft personalised emails
  Key location    : GROQ_API_KEY env var (run_pipeline.bat / Railway env vars)

  FRONTEND — REACT
  ────────────────
  Framework       : React 19.0
  Bundler         : Vite 4.5        — dev server + production build (Node 16 compat)

  Styling         : Plain inline CSS — no external UI library
  HTTP            : fetch API       — built into the browser
  Dev proxy       : Vite proxy      — /api/* → localhost:8000 (no CORS in dev)

  DATA FLOW — HOW A JOB MOVES THROUGH THE SYSTEM
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  STATUS STATES (stored in jobs.db)
  ──────────────────────────────────
  new      → Job fetched from Indeed, not yet scored
  scored   → (intermediate — Groq gave it a score)
  pending  → Score >= threshold, email drafted, waiting for your review
  approved → You clicked Approve in the UI (used if email sending failed)
  sent     → Email successfully delivered via Gmail SMTP
  skipped  → You clicked Skip in the UI, or score was below threshold
  error    → Groq or API error during processing (see error_message column)

  PIPELINE TRIGGER OPTIONS
  ─────────────────────────
  1. Click "▶ Run Pipeline" button in the web UI  ← easiest
  2. Double-click run_pipeline.bat  (also sets up Task Scheduler)
  3. POST http://localhost:8000/api/run  (direct API call)

  PIPELINE FLOW
  ──────────────

  [UI button / run_pipeline.bat / Task Scheduler]
        │
        ▼
  api/main.py → POST /api/run
  Spawns background thread (non-blocking) → returns immediately
        │
        ▼
  main.py: run_pipeline()
  1. Check Groq API key is set         → abort with clear message if not
  2. Validate Gmail credentials        → warn only, don't abort
  3. scraper.py: fetch_all_queries()
     ├─ Search 4 queries × 2 pages = ~120 API calls
     ├─ Filter: remote_work_model contains "remote"
     ├─ Deduplicate by job_id (INSERT OR IGNORE)
     └─ Fetch full descriptions only for NEW jobs (API budget guard)
  4. storage.py: insert_jobs()         → new jobs get status = 'new'
  5. For each new job:
     ├─ scorer.py: score_job()
     │   └─ Groq (llama-3.3-70b-versatile): compare job vs RESUME_TEXT
     │       Returns: {"score": 8, "reasoning": "...", "key_matches": [...]}
     │       Speed: <1 second per job
     ├─ If score >= 7:
     │   ├─ scorer.py: draft_email()
     │   │   └─ Groq: write personalised 150-200 word outreach email
     │   └─ status → 'pending'
     └─ If score < 7: status → 'skipped'
  6. log_run() → writes stats to run_log table
  7. Pipeline thread sets last_result = "success"

  UI POLLING WHILE PIPELINE RUNS
  ────────────────────────────────
  App.jsx polls GET /api/run/status every 3 seconds
  → Shows spinner + "Fetching jobs & scoring with Groq AI..."
  → When running=false: auto-refreshes jobs list and stats
  → Shows "Pipeline finished — jobs updated!" for 5 seconds

  AI PROMPTS USED
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  SCORING PROMPT  (src/scorer.py → _SCORING_PROMPT)
  ──────────────────────────────────────────────────
  Input  : Your RESUME_TEXT + job title, company, location, type, description
  Output : JSON → { score: int(1-10), reasoning: str, key_matches: [], key_gaps: [] }
  Model  : llama-3.3-70b-versatile  |  temperature: 0.3  |  max_tokens: 600
  Format : response_format={"type": "json_object"}  (guaranteed valid JSON)

  Scoring scale:
    1–3  Poor fit   (wrong stack, wrong level, major skill gaps)
    4–6  Partial    (some overlap, significant gaps)
    7–8  Good fit   (strong skill overlap, right seniority)
    9–10 Excellent  (near-perfect match)

  EMAIL DRAFTING PROMPT  (src/scorer.py → _EMAIL_PROMPT)
  ────────────────────────────────────────────────────────
  Input  : Your RESUME_TEXT + job title, company, description
  Output : JSON → { subject: str, body: str }
  Model  : llama-3.3-70b-versatile  |  temperature: 0.6  |  max_tokens: 600

  Email rules enforced in prompt:
    • 150–200 words maximum
    • Opens with something specific to this company/role
    • Highlights 2–3 resume achievements relevant to THIS job
    • Ends with a low-pressure call to action
    • Sounds human, not templated

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  "GROQ_API_KEY not set" / "Groq API error"
    → Open run_pipeline.bat and add your key:  set GROQ_API_KEY=gsk_...
    → Get a free key at https://console.groq.com → API Keys → Create API Key

  "Groq rate limit exceeded"
    → Free tier allows 14,400 req/day — very unlikely to hit this
    → If it happens, the pipeline retries automatically; or wait 1 hour

  "Gmail authentication failed"
    → App Password must be exactly 16 characters with no spaces
    → Go to Google Account → Security → App Passwords to regenerate

  "Indeed API 500 error"
    → Temporary outage on omkar.cloud's side (they fix within 24–48h)
    → Your key is valid — just re-run the pipeline the next day

  "Indeed API 401 error"
    → API key invalid or expired
    → Get a new key at https://omkar.cloud/indeed-scraper/

  "0 remote jobs found (key valid, no errors)"
    → Increase INDEED_MAX_PAGES in config/settings.py
    → Or broaden REMOTE_KEYWORDS to include "hybrid"

  "Pipeline button spins but nothing happens"
    → Check uvicorn logs in Terminal 1 for errors
    → Verify GROQ_API_KEY is set in the terminal where uvicorn runs

  "npm run dev fails"
    → Requires Node.js 16+. Check: node --version
    → Re-install Node.js from https://nodejs.org
